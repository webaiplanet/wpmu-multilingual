### 已确认风险

1. 新同步在关系目标缺失时仍会自动认领相同 ID 的目标文章。
2. 相同 ID 未命中后仍会自动认领相同 slug 和 post type 的目标文章。
3. 更新、删除、恢复、翻译写回和前台链接没有统一的目标身份校验入口。
4. 永久删除策略为 `all` 时，错误关系可能导致无关目标文章被永久删除。
5. “重建关联”会先清空关系表，再使用 meta、slug 和 ID 猜测重建，不适合当前生产数据。
6. 关系表未限制一个 `(target_blog_id, target_post_id)` 只能属于一个源文章。

## 4. 兼容性约束

- 不修改历史文章 ID、URL、slug、发布时间和发布状态。
- 不批量重建、重新同步或覆盖现有文章。
- 已存在且无冲突的历史关系继续有效，即使目标文章暂时没有来源 meta。
- 历史关系正常参与下一次受控同步时，可以补写缺失的来源 meta。
- 来源 meta 与关系表冲突时必须停止写入并记录异常，不能自动选择任一方覆盖。
- 新文章必须使用 `wp_insert_post()` 返回的实际目标 ID 建立关系。

## 5. 验证记录

### 2026-07-21 / 0.9.8.7 PHP 字段增量同步

- 新增源字段哈希快照，覆盖标题、摘要、正文和 Meta Key；只记录哈希和字段键，不复制整篇正文快照。
- 新增任务变化清单与真实翻译完成证据，关系状态不再单独决定译文保护。
- 未翻译目标同步变化源字段；已有译文目标保留未变化译文，并以“更新已翻译的内容”状态进入增量队列。
- OpenAI、OpenCC、Agent 只翻译和写回变化清单中的字段；无变化字段不发送、不覆盖。

### 2026-07-24 / 0.9.8.8 taxonomy term 本体同步

- 新增 `WPMU_ML_Core_Term_Sync_Trait`，监听源站 `created_term`、`edited_term`、`delete_term`。
- 只处理 `sync_taxonomies` 白名单内 taxonomy；目标站和分站事件不反向同步。
- 新增/编辑时先查 term 关系，关系缺失时只针对当前 term 尝试同 slug + taxonomy、同 ID + taxonomy 认领并修复关系。
- 父子 taxonomy 先同步父级，目标 parent 使用目标站父级 term ID。
- 删除时只按关系表删除目标 term 并清理关系，不按 slug 模糊删除。
- 文章同步分类关系改为源 term ID 映射目标 term ID，避免 term ID 不一致时挂错目标分类或标签。
- 任务表迁移仅新增 `change_manifest`、`translated_content`，不重建关系、不修改文章 ID 或历史 URL。
- 两组事务回滚测试通过：未翻译目标只同步变化 Meta 并进入 `pending`；已翻译目标保留旧译文并进入 `translated_update_pending`；Agent Payload 均只包含变化字段，测试残留为 0。

### 2026-07-24 / 0.9.8.11 OpenCC 源语言保护

- OpenCC 自动路由从“目标是繁体中文即可 fallback”改为“源站为简体中文且目标为繁体中文才 fallback”。
- 后台目标语言规则中，只有符合“简体中文源站 → 繁体中文目标站”时才显示 OpenCC 选项。
- 非中文源站公开安装插件时，不会因为目标语言是繁体中文而误进入简繁转换。
- OpenCC 环境检测的必需判断同步收紧，避免误报。

### 2026-07-24 / 0.9.8.10 后台未发布语言配置清理

- 移除“内容类型”页里旧的“隐藏未发布语言”checkbox，避免和“语言切换”页重复。
- “语言切换”页保留唯一入口，并改名为“未发布语言处理”。
- hreflang 固定只输出已发布且可索引的目标文章；语言切换器可单独选择隐藏未发布语言或显示提示弹窗。
- 修正 term name 翻译写入被源站名称覆盖的问题。

### 2026-07-24 / 0.9.8.9 taxonomy term 翻译开关

- 新增 `translate_term_name` 和 `translate_term_description` 设置，默认关闭。
- 后台“内容类型”页新增分类/标签本体翻译开关。
- term 同步时可按目标语言翻译 `name` / `description`，`slug` 仍固定同步源站。
