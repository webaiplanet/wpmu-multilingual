# WPMU 多语言插件 0.9.8.14 WPMU 语言切换器显示名称报告

## 问题

菜单调用功能在后台和 WordPress “外观 → 菜单”中显示为 `Language Switcher`。其他多语言插件也可能使用同名面板，容易让管理员误选。

## 处理结果

- 后台设置页文案改为 `WPMU 语言切换器`。
- `register_post_type()` 的显示标签改为 `WPMU 语言切换器`。
- “外观 → 菜单”中的菜单项面板名称会显示为 `WPMU 语言切换器`。
- 内部 post type 标识仍为 `wpmu_ml_switcher`，不改数据库对象类型，避免影响已有菜单项。

## 验收建议

- 进入“翻译设置 → 语言切换 → 菜单调用”，确认说明显示 `WPMU 语言切换器`。
- 进入任意语言站点的“外观 → 菜单”，确认左侧菜单项面板显示 `WPMU 语言切换器`。
- 确认已有菜单项仍可正常显示和保存。
