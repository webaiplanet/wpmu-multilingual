# WPMU 多语言插件 0.9.8.7 PHP 字段增量同步测试报告

## 处理目标

解决关系状态为 `needs_update` / `review_required` 时把从未翻译的目标误判为已有译文的问题，并实现源文章更新后的字段级同步和增量翻译。

## 当前规则

```text
源文章保存
→ PHP 对标题、摘要、正文和 Meta Key 计算哈希差异
→ 未翻译目标同步变化源字段，任务进入 pending
→ 已翻译目标保留现有译文，任务进入 translated_update_pending
→ OpenAI / OpenCC / Agent 只翻译并写回变化清单中的字段
→ 成功后记录真实翻译完成证据
```

旧任务没有变化清单时继续使用全量翻译，不改变历史任务兼容行为。

## 数据库变更

翻译任务表新增：

- `change_manifest LONGTEXT NULL`
- `translated_content TINYINT(1) NOT NULL DEFAULT 0`

升级只新增字段，并从明确完成状态补写翻译证据；不重建关系、不修改文章 ID、slug、URL 或正文。

## 回滚测试

测试脚本：`tests/incremental-sync-smoke.php`

### 从未翻译目标

- 源站只修改一个测试 Meta。
- 目标标题和正文与源站相同。
- 结果：只同步该 Meta，标题和正文不重复处理。
- 任务状态：`pending`。
- Agent Payload：仅包含该 Meta 字段。

### 已翻译目标

- 目标标题、正文和 Meta 使用模拟译文。
- 源站只修改一个测试 Meta。
- 结果：目标标题、正文和旧 Meta 译文保持不变。
- 任务状态：`translated_update_pending`，后台显示“更新已翻译的内容”。
- 变化清单和 Agent Payload：仅包含该 Meta 字段。

两组测试均在数据库事务内执行并回滚，脚本包含硬断言。测试后源/目标测试文章残留均为 0。
